====== О проекте Lede ======

===== Перезагрузка сообщества OpenWRT =====

Проект Lede основан как спин-офф проекта OpenWRT и преследует во многом те же самые целей. Мы разрабатываем Embedded Linux, что делает его легким для разработчиков, системных администраторов и других энтузиастов Linux, чтобы легко настроить программное обеспечение для встраиваемых устройств, особенно беспроводных маршрутизаторов. Имя // Lede // расшифровывается как  // Linux Embedded Development Environment //.

Большинство члены проекта это значительная доля наиболее активных членов сообщества OpenWRT. Мы намерены вдохнуть новую жизнь в развития Embedded Linux  путем создания сообщества с сильным акцентом на прозрачность, сотрудничества и децентрализацию.

Цели проекта LEDE’s 

  * Создать Embedded Linux дистрибутив с акцентом на стабильность и       функциональность.
  * Регулярно создавать и улучшать стабильность системы на основе обратной связью с сообществом тестирующих LEDE на своих устройствах.
  * Создание прозрачных процессов принятия решений с участием широкого круга общественности и общественных собраний.

Мы решили создать этот новый проект из-за множества вопросов, которые мы не могли решить или реализовать в рамках проекта OpenWrt / сообщества:

  -  Количество активных разработчиков ядра низкое, нет процесса привлечения новых людей к разработке.
  -  Ненадежная инфраструктура, меры, внутренние разногласия и неудачи.
  -  Нехватка людей с постоянным доступа для обработки входящего потока патчей, слишком мало внимания к тестированию и регулярной сборки.
  -  Отсутствие внимания к стабильности и документации.

Для решения этих проблем мы создали проект Lede по-другому с сравнению OpenWrt:

  -  Все наши каналы связи являются открытыми, некоторые только для чтения, для тех кто не является членам сообщества, чтобы исключить возникновения помойки.
  -  Наш процесс принятия решений является более открытым, с приблизительностью  50/50 разработчиков и опытных пользователей, имеющих право голоса.
  -  Наша инфраструктура в многом упрощена.
  -  Мы сделали нашу политику слияния более либеральной, основываясь на нашем опыте с использования  пакета GitHub в OpenWRT.
  -  У нас есть сильный акцент на автоматизацию тестирования в сочетании с процессом упрощенного выпуска.
